Comment accéder à Claude Design et faire ton premier prototype
🚀 Démarrage
Le guide étape par étape pour accéder à Claude Design et générer ton premier prototype en moins de 5 minutes.
ÉTAPE 1 — Accès → Va sur claude.ai/design → Disponible pour les abonnés Pro, Max, Team et Enterprise → Si tu es sur le bon plan, le menu "Design" apparaît dans la barre de navigation ÉTAPE 2 — Crée ton premier projet → Clique sur "New Design Project" → Donne un nom à ton projet (ex : "App méditation", "Landing page portfolio") ÉTAPE 3 — Décris ce que tu veux en langage normal → Pas besoin de jargon technique → Exemple : "Crée-moi une app mobile de méditation, couleurs nature (vert sauge, beige), typographie zen et épurée, interface minimaliste" → Claude génère un prototype complet et cliquable ÉTAPE 4 — Ajuste avec précision → Clique sur n'importe quel élément pour le sélectionner → Modifie les textes directement à la main → Utilise les curseurs pour couleurs, espacement, typographie en temps réel → Commente une zone précise pour que Claude la modifie ÉTAPE 5 — Exporte ou connecte à Claude Code → Bouton "Send to Claude Code" pour passer directement au développement → Ou exporte en : PDF · HTML · PowerPoint · Canva → Partage un lien cliquable directement depuis l'interface
5 prompts Claude Design qui donnent les meilleurs résultats
✍️ Prompts
Les structures de prompts testées pour générer des designs propres du premier coup — pour 5 cas d'usage différents.
── APP MOBILE ── "Crée une app mobile [type d'app]. Style visuel : [minimaliste / coloré / corporate / dark mode]. Couleurs principales : [couleurs]. L'écran principal doit montrer [contenu clé]. Navigation en bas avec 4 onglets : [liste]." ── LANDING PAGE ── "Crée une landing page pour [produit/service]. Cible : [audience]. Ton : [professionnel / amical / urgent]. Structure : hero avec accroche + 3 bénéfices + témoignages + CTA final. Couleur dominante : [couleur]." ── DASHBOARD ── "Crée un dashboard pour [contexte : e-commerce / SaaS / analytics]. Affiche en priorité : [métrique 1], [métrique 2], [métrique 3]. Style : dark mode / light mode. Inclus un graphique [type] et une table de données." ── PRÉSENTATION ── "Crée une présentation de [X] slides sur [sujet]. Slide 1 : titre accrocheur. Slides 2-[X-1] : [structure souhaitée]. Slide finale : CTA / conclusion. Style : [corporate / créatif / minimaliste]. Couleurs : [palette]." ── RÉSEAUX SOCIAUX ── "Crée un template de post Instagram [format carré/story] pour [marque/sujet]. Style cohérent avec [description de la charte]. Texte principal : [message]. Inclus un espace pour logo en [position]."
Le workflow complet : de l'idée au code en une conversation
⚡ Workflow
Le process exact pour passer de zéro à un prototype fonctionnel et codé sans changer d'outil — utilisé par les équipes qui compressent une semaine de travail en une conversation.
PHASE 1 — Brief (2 min) Avant d'ouvrir Claude Design, réponds à ces 4 questions : • Quel est l'objectif principal de ce projet ? • Qui sont les utilisateurs ? • Quels sont les 3 écrans/sections indispensables ? • As-tu une charte graphique existante ? (couleurs, typos, logo) PHASE 2 — Génération du prototype (5-10 min) 1. Ouvre claude.ai/design → New Project 2. Colle ton brief complet en une seule fois 3. Si tu as une charte : upload tes fichiers (CSS, PDF brand guide, captures) → Claude applique automatiquement tes couleurs et typos 4. Laisse Claude générer sans l'interrompre 5. Fais une première passe de retours en commentant les zones précises PHASE 3 — Itération (5-10 min) • Pour modifier : "Change la couleur du bouton CTA en [couleur]" • Pour restructurer : "Déplace le bloc témoignages avant le pricing" • Pour le ton : "Rends le texte du hero plus direct et percutant" • Utilise les curseurs visuels pour les ajustements fins PHASE 4 — Vers le code (1 clic) • Si tu as Claude Code : bouton "Send to Claude Code" → De l'idée au prototype au code, sans changer d'outil • Sans Claude Code : → Export HTML pour intégrer directement → Export Canva pour retouches en équipe → Lien de partage pour validation client RÉSULTAT : ce qui prenait 1 semaine avec Figma + dev → 1 conversation.
